filters.py — the most important file. All four filters run in one pass, zero tokens:

  • fix_linkedin_url() handles both search URLs (?currentJobId=XYZ) and already-correct view URLs, normalising everything to /jobs/view/<id>/
  • classify_job_type() checks employment_type field first, then title regex — so it works across Remotive, Greenhouse, LinkedIn etc.
  • Seniority gate runs after classification, so an "Intern" titled "Senior Intern" (rare but exists) still passes
  • extract_required_experience() uses 5 regex patterns covering the common English phrasings; takes the minimum when a range like "3–5 years" appears

crew.py — the critical pipeline reorder. apply_all_filters() is now called inside fetch_all_jobs_tool, right after dedup, before any Groq call is made. The old pipeline filtered after scoring; this one filters before. With 25 fresh jobs → ~11 after filters, Groq only processes 11 instead of 25 — roughly 40% fewer tokens, and higher-quality inputs.

utils.py — dedup now runs on two axes: job_id (same source) plus a (title, company) MD5 fingerprint. This catches the same role appearing on both LinkedIn and Greenhouse on the same day.

reporter.py — build_html_report() takes two pre-split lists. Each Apply button uses job["url"] directly, which is already the canonical link because fix_linkedin_url() ran in filters.py before anything else touched the data.
